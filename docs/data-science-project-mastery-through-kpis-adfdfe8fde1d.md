**项目控制**在数据科学项目中至关重要，因为所需时间和成本往往难以估算。客户希望利用机器学习方法高效且成功地实施他们的用例。通过有意义的**关键绩效指标（KPIs）**，可以积极影响项目的持续控制。据一些[研究](https://designingforanalytics.com/resources/failure-rates-for-analytics-bi-iot-and-big-data-projects-85-yikes/)显示，85%的数据科学项目失败，这要求及早识别障碍。

在传统的 IT 项目中，存在许多用于项目控制的 KPIs，但这些对于数据科学项目来说并不充分。因此，在本文中，我们介绍了 IT 项目的基本 KPIs，并分析了它们在数据科学项目中的适用性。结果将表明，特定的 KPIs 可以使数据科学项目的项目控制更加透明。

文章首先处理了项目控制的基础知识，并介绍了一些来自软件行业的关键绩效指标（KPIs）。接下来，我们使用这些呈现的 KPIs，并根据数据科学项目使用过程模型 CRISP-DM（**CR**oss-**I**ndustry **S**tandard **P**rocess for **D**ata **M**ining）对其进行分析。分析还包括了来自数据科学项目的个人项目经验。
